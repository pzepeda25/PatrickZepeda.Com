## 2024-10-27 - [Avoid State Updates in requestAnimationFrame]
**Learning:** Calling React state setters (like `useState`) directly inside `requestAnimationFrame` loops completely circumvents performance if the state isn't actually being read by the component's JSX tree or needs to trigger a full re-render on every frame. It effectively causes the entire component to be re-rendered ~60 times a second, which is terrible for performance.
**Action:** Always prefer `useRef` for tracking frequently updating animation values (like speed, position, etc.) unless the component's UI directly depends on the updated value in a way that necessitates React's render lifecycle. If the UI does depend on it, carefully consider if the update can be debounced or separated into a smaller component so only that small component re-renders.
