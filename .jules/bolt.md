## 2025-04-02 - Unused React State in `requestAnimationFrame` Loops
**Learning:** Calling a React state setter (like `useState`) inside a `requestAnimationFrame` loop, even if the value itself isn't used in the component's JSX, forces React to re-render the component at 60 frames per second. This is a severe performance anti-pattern that burns CPU cycles for no visual output. In `src/components/ui/scanner-card-stream.tsx`, the `speed` state was being set 60 times a second but never displayed.
**Action:** Audit high-frequency render loops (like `requestAnimationFrame`, `setInterval`, or canvas rendering loops) to ensure no React state mutations occur within them unless strictly necessary for UI updates. Use `useRef` for values that need to be tracked inside these loops but don't need to trigger a render.
